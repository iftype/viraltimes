# Quiz feature

- 목적: 사용자가 밈을 이미 아는지와 ViralTimes의 설명을 본 뒤 이해에 도움이 됐는지를 분리 측정한다.
- 한 세션은 published 사전 전체에서 서로 다른 5개 항목을 매번 무작위로 보여준다. 관리자 카드 설정은 선택 항목의 분야 라벨에만 사용한다.
- 판단 응답은 `know`/`dont_know`, 실제 상세 modal은 `view_media`, 완료 후 밈·서비스 이동은 `open_meme`/`open_service`다.
- 각 테스트 시작마다 `runId`를 발급하고 `start` → 카드별 step → `complete` → 외부 CTA 이벤트를 저장해 중도 이탈 퍼널을 계산한다.
- 카드 판단 뒤 고정된 4개 경험 항목을 한 줄씩 복수선택하는 체크리스트를 먼저 표시한다. 선택 없음도 제출로 기록하며, 이후 관리자가 설정한 최대 5개의 객관식 추가 설문을 순서대로 표시한다. 모든 설문 응답을 저장한 뒤에만 `complete`를 기록한다.
- 원시 이벤트는 보존하되 통계의 인지도는 같은 익명 세션·카드의 마지막 판단만 반영한다.
- 브라우저 localStorage의 무작위 세션 ID만 사용하며 이름·이메일·계정 정보와 연결하지 않는다.
- 사전 대표 썸네일이 없으면 원본·바이럴 영상의 저장된 썸네일을 보완해 사용한다. 이미지가 없거나 로드에 실패하면 제목과 accent 기반 placeholder를 보여주고, 설명이 없으면 수집 중 문구를 사용한다.
- 퀴즈 카드와 누적 수치는 공개된 서버 데이터만 사용하며, API 실패 시 가짜 샘플 통계를 표시하지 않는다.
- 모바일 퀴즈는 헤더·푸터를 숨기고 `100dvh` 안에서 카드와 조작 버튼을 모두 보여 페이지 스크롤·수직 drag를 만들지 않는다. 카드는 수평 swipe만 반응한다. 768px 이상에서는 430px 폭의 모바일 프레임을 중앙에 두고 같은 모바일 정보 밀도와 interaction을 유지하며, 넘긴 카드는 프레임 밖에서 노출하거나 DOM에 남기지 않는다.
- 영상이 있는 카드는 썸네일 위 명시적인 재생 버튼을 누르면 카드 안에서 YouTube·TikTok·Instagram embed를 생성한다. embed 조작은 카드 drag와 분리하고, 아래 `유래·뜻 자세히 보기`는 실제 `/memes/:slug` 페이지를 modal iframe으로 연다. 알아요·몰라요는 불투명한 고대비 버튼을 사용한다.
- 첫 카드에는 좌우 방향과 알아요·몰라요 의미를 겹쳐 보여주고 첫 drag 또는 판단 뒤 즉시 숨긴다. 상세 modal은 모바일 84dvh, desktop 82vh로 배경 맥락이 남게 한다.
- 완료 화면은 5개 항목의 `더 알아보기`, 다른 참여자 인지도, `테스트 다시 하기`와 `서비스 보러 가기`를 한 화면에 제공한다.
