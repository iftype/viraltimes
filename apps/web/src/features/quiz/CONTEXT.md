# Quiz feature

- 목적: 사용자가 밈을 이미 아는지와 ViralOrigin의 설명을 본 뒤 이해에 도움이 됐는지를 분리 측정한다.
- 한 세션은 관리자가 정한 최대 5개 published 사전 항목을 보여준다. deck이 비어 있을 때만 공개 항목을 무작위로 고른다.
- 판단 응답은 `know`/`dont_know`, 실제 상세 modal은 `view_media`, 완료 후 밈·서비스 이동은 `open_meme`/`open_service`다.
- 각 테스트 시작마다 `runId`를 발급하고 `start` → 카드별 step → `complete` → 외부 CTA 이벤트를 저장해 중도 이탈 퍼널을 계산한다.
- 원시 이벤트는 보존하되 통계의 인지도는 같은 익명 세션·카드의 마지막 판단만 반영한다.
- 브라우저 localStorage의 무작위 세션 ID만 사용하며 이름·이메일·계정 정보와 연결하지 않는다.
- 썸네일이 없으면 accent 기반 placeholder를 보여주고, 설명이 없으면 수집 중 문구를 사용한다.
- 퀴즈 카드와 누적 수치는 공개된 서버 데이터만 사용하며, API 실패 시 가짜 샘플 통계를 표시하지 않는다.
- 모바일 퀴즈는 헤더·푸터를 숨기고 `100dvh` 안에서 카드와 조작 버튼을 모두 보여 페이지 스크롤·수직 drag를 만들지 않는다. 카드는 수평 swipe만 반응한다.
- 카드의 `영상 보러 가기`는 실제 `/memes/:slug` 페이지를 modal iframe으로 연다. 알아요·몰라요는 불투명한 고대비 버튼을 사용한다.
- 첫 카드에는 좌우 방향과 알아요·몰라요 의미를 겹쳐 보여주고 첫 drag 또는 판단 뒤 즉시 숨긴다. 상세 modal은 모바일 84dvh, desktop 82vh로 배경 맥락이 남게 한다.
- 완료 화면은 5개 항목의 `더 알아보기`, 다른 참여자 인지도, `테스트 다시 하기`와 `서비스 보러 가기`를 한 화면에 제공한다.
