# Admin app context

- `iftype.store/viral`에서 제공하는 독립 정적 Next.js 앱이다.
- `basePath`는 `/viral`이며 기존 iftype.store 앱과 정적 자산 경로가 겹치지 않는다.
- 브라우저는 같은 출처의 `/viral/api/v1/admin/*`만 호출하고 Nginx가 ViralOrigin API로 전달한다.
- 인증 비밀이나 운영 데이터는 번들에 포함하지 않는다.
- 운영 서버에서는 정적 파일만 제공하므로 별도 Node 프로세스와 상시 메모리가 필요 없다.
- 사전 관리 탭은 draft/published/archived 항목을 API로 작성·수정하며 공개 저장 즉시 클라이언트에 반영한다.
- 사전 목록은 검색·상태·종류 필터와 선택 UI를 제공한다. draft·archived는 즉시 삭제하고, published가 포함된 삭제만 확인하며 선택 삭제는 별도 버튼으로 처리한다.
- 코리아 마이너 밈은 설명·원본·타임라인을 비워도 되며 복수 커뮤니티 링크를 저장한다.
- 원본·바이럴·원곡·안무 원본 링크 자동 채우기는 관리자 전용 metadata preview API를 사용한다. YouTube/TikTok oEmbed와 Instagram Open Graph를 우선하며 AI 결과를 자동 공개하지 않는다.
- 카테고리 탭은 서버 category의 표시 이름, slug, 설명, 정렬 순서와 활성 상태를 관리한다. 사전 항목은 category ID를 복수 선택한다.
- 상세 페이지에서 등록된 수정 제안은 `수정 제안` inbox 탭에 별도로 표시한다.
- 매치 로그 탭은 익명 브라우저별·카드별 행동을 묶어 시작·종료 시각, 진행 시간, 완료 여부와 실행별 시작→1~5번→완료→서비스 유입 퍼널을 보여준다. 상세 modal·밈 상세·서비스 destination도 집계하며 참여자 기록 하나 또는 전체 로그를 삭제하고 CSV로 내려받을 수 있다.
- 퀴즈 카드 탭은 공개 사전에서 최대 5개를 골라 분야와 출제 순서를 저장한다.
- 사전 편집의 연결·파생 밈 선택은 `relatedMemeIds` 관계를 관리한다.
- 사전 편집의 `바이럴 영상 TOP3`는 영상별 피드 노출 여부를 관리하며 기존 `trendingVideos` 데이터 키를 유지한다. 챌린지 원본도 피드 노출을 선택하고 원곡·안무 원본은 상세용 보조 영상으로 별도 관리한다.
- 원본 영상과 원본 뜻·유래 설명은 선택이다. 원본 영상이 없으면 API가 검토 상태를 `needs-review`로 저장한다.
